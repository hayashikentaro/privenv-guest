# Manifest Spec

`privenv.manifest.json` is the only file the Guest reads.

It is generated by the Host and is safe for Guest consumption. It may include capability metadata and env variable names, but it must never include env values, vault values, tokens, PII, OAuth tokens, SSH keys, browser sessions, or raw secrets.

`privenv-guest` validates the manifest shape with shared validators from `@privenv/protocol`. Loading `privenv.manifest.json` from the current working directory remains Guest-specific local behavior.

## Shape

```ts
interface GuestManifest {
  version: string;
  capabilities: ManifestCapability[];
}

interface ManifestCapability {
  id: string;
  kind: "command";
  description: string;
  command?: {
    program: string;
    args: string[];
  };
  env: ManifestEnvReference[];
}

interface ManifestEnvReference {
  name: string;
  source?: "secret";
  exposedToGuest: false;
}
```

## Rules

- `id` is the value the Guest uses as `EffectRequest.capabilityId`.
- `description` is safe display text.
- `command` is metadata only when the Host declares it safe.
- `env.name` may appear because names are not values.
- `exposedToGuest` must be `false`.
- no field may contain a raw secret value.
